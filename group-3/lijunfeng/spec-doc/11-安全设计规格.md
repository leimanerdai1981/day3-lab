# 11-安全设计规格

## 1. 安全目标

| 目标 | 说明 |
|------|------|
| 数据完整性 | 确保基金数据和局势数据不被篡改 |
| 服务可用性 | 防止恶意请求导致服务不可用 |
| 数据保密性 | 保护数据源配置信息 |

← 07 §4 安全约束

## 2. 认证授权

### 2.1 认证策略

| 项目 | 说明 |
|------|------|
| 认证方式 | 无需认证 |
| 理由 | Demo 应用，内部使用 |
| 未来扩展 | 如需扩展，可添加 JWT 认证 |

### 2.2 访问控制

| 资源 | 访问权限 |
|------|----------|
| 前端页面 | 公开访问 |
| API 接口 | 公开访问，限流保护 |
| 数据文件 | 仅后端服务读取 |

## 3. 数据安全

### 3.1 数据传输

| 项目 | 说明 |
|------|------|
| 传输协议 | HTTP（Demo 阶段） |
| 未来扩展 | 生产环境使用 HTTPS |

### 3.2 数据存储

| 项目 | 说明 |
|------|------|
| 敏感数据 | 无敏感用户数据 |
| 配置文件 | 数据源配置存储在环境变量 |
| 日志脱敏 | 日志中不包含敏感信息 |

## 4. 输入安全

### 4.1 参数校验

| 参数类型 | 校验规则 |
|----------|----------|
| week | 整数，范围 1-100 |
| limit | 整数，范围 1-20 |
| 字符串 | 长度限制，防注入 |

← 09 §5 参数校验规则

### 4.2 防注入措施

| 措施 | 说明 |
|------|------|
| SQL 注入 | 无数据库，使用 JSON 文件 |
| XSS 防护 | 前端 React 自动转义 |
| 路径遍历 | 文件访问限制在 data/ 目录 |

## 5. 安全监控

| 监控项 | 说明 |
|--------|------|
| 请求日志 | 记录 API 请求和响应时间 |
| 错误日志 | 记录异常和错误信息 |
| 限流监控 | 监控请求频率，防止滥用 |

## 6. 安全应急响应

| 场景 | 应对措施 |
|------|----------|
| 数据源异常 | 切换到本地缓存数据 |
| 服务异常 | 重启服务，检查日志 |
| 数据损坏 | 从备份恢复 JSON 文件 |
