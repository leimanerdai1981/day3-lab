## 2. 技术选型

### 2.1 前端

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| React | 18.x | 组件化开发，生态成熟 |
| Vite | 5.x | 构建速度快，配置简单 |
| Tailwind CSS | 3.x | 原子化 CSS，快速开发 |
| Recharts | 2.x | 图表展示，时间轴可视化 |

### 2.2 后端

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Python | 3.11+ | 数据处理能力强，RSS 抓取生态好 |
| FastAPI | 0.100+ | 高性能异步框架，自动生成文档 |
| Uvicorn | 0.24+ | ASGI 服务器，支持异步 |
| Pydantic | 2.x | 数据校验，类型安全 |

### 2.3 数据存储

| 存储 | 用途 | 选型理由 |
|------|------|----------|
| JSON 文件 | 基金数据、局势数据 | Demo 阶段简单够用，无需数据库 |
| 内存缓存 | API 响应缓存 | 减少重复计算，提升性能 |

## 3. 模块划分

### 3.1 前端模块

| 模块 | 职责 | 对应 US |
|------|------|---------|
| TimelineView | 时间轴展示、周节点切换 | US-001, US-004 |
| ThematicFunds | 主题基金卡片展示 | US-002 |
| SmallScaleFunds | 精选小规模基金展示 | US-003 |
| SituationScore | 局势分值可视化 | US-001 |

### 3.2 后端模块

| 模块 | 职责 | 对应 API |
|------|------|----------|
| timeline_service | 时间轴数据管理 | /api/timeline |
| fund_service | 基金数据筛选 | /api/funds/* |
| situation_service | 局势分值获取 | /api/situation/score |
| rss_fetcher | RSS 数据抓取 | 内部服务 |

## 4. 外部依赖

| 依赖 | 用途 | 备选方案 |
|------|------|----------|
| Polymarket API | 获取停战概率分值 | 手动配置分值数据 |
| RSS 源 | 获取美伊战争新闻 | 本地新闻数据文件 |
