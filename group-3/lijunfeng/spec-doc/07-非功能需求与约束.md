# 07-非功能需求与约束

## 1. 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首屏加载时间 | ≤ 3 秒 | 从请求到页面可交互 |
| API 响应时间 | ≤ 500ms | 95% 的请求 |
| 时间轴切换响应 | ≤ 200ms | 周节点切换后数据更新 |
| 并发用户数 | ≥ 10 | Demo 阶段支持 |

← 04 §6 非功能需求（简述）

## 2. 可用性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 系统可用性 | ≥ 99% | Demo 阶段，非生产环境 |
| 数据更新频率 | 每日 1 次 | RSS 数据和局势分值 |
| 故障恢复时间 | ≤ 5 分钟 | 手动重启服务 |

## 3. 兼容性需求

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Firefox 88+, Safari 14+ |
| 设备 | 桌面端为主，支持平板 |
| 分辨率 | 最低 768px 宽度 |

← 03 §3 范围边界：仅 Web 端

## 4. 安全约束

| 约束 | 说明 |
|------|------|
| 无认证要求 | Demo 应用，无需登录 |
| 数据本地存储 | 基金数据存储在本地 JSON 文件 |
| API 限流 | 每分钟最多 100 次请求 |

→ 11 安全设计规格（详细）

## 5. 数据约束

| 约束 | 说明 |
|------|------|
| 数据保留期 | 保留 3 月初至今的数据 |
| 数据备份 | 无需自动备份，手动管理 JSON 文件 |
| 数据一致性 | 周节点数据与基金数据通过 weekNumber 关联 |

## 6. 开发约束

| 约束 | 说明 |
|------|------|
| 开发周期 | Demo 阶段，快速迭代 |
| 代码规范 | 遵循团队基础规范 |
| 测试要求 | 核心功能手动测试 |
| 文档要求 | 关键逻辑添加代码注释 |

← 03 §4 成功标准

## 7. 外部依赖约束

| 依赖 | 约束 |
|------|------|
| Polymarket | 分值数据可能延迟，需有 fallback |
| RSS 源 | 网络不稳定时可能获取失败 |
| 基金数据 | 依赖本地 JSON 文件，需定期更新 |
