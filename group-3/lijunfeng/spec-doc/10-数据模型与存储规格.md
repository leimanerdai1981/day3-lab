# 10-数据模型与存储规格

## 1. 实体定义

### 1.1 WeekNode（周节点）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| weekNumber | integer | 是 | 周编号，从 1 开始 |
| startDate | string | 是 | 周开始日期，格式 YYYY-MM-DD |
| endDate | string | 是 | 周结束日期，格式 YYYY-MM-DD |
| situationScore | integer | 是 | 局势分值，0-100 |
| summary | string | 是 | 局势动态摘要 |
| createdAt | string | 是 | 创建时间，ISO 8601 |
| updatedAt | string | 是 | 更新时间，ISO 8601 |

← 09 §4.1, 4.2 响应 DTO

### 1.2 ThematicFund（主题基金）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| theme | string | 是 | 主题名称：原油、黄金、煤炭、电力、天然气 |
| fundName | string | 是 | 基金名称 |
| fundCode | string | 否 | 基金代码 |
| returnRate | number | 是 | 涨跌幅，百分比 |
| weekNumber | integer | 是 | 所属周编号 |

← 09 §4.3 响应 DTO

### 1.3 SmallScaleFund（小规模基金）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| fundName | string | 是 | 基金名称 |
| fundCode | string | 否 | 基金代码 |
| company | string | 是 | 基金公司：广发、易方达、富国、南方、华夏 |
| scale | number | 是 | 基金规模，单位：亿 |
| returnRate | number | 是 | 涨跌幅，百分比 |
| weekNumber | integer | 是 | 所属周编号 |

← 09 §4.4 响应 DTO

### 1.4 SituationData（局势数据）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| score | integer | 是 | 局势分值，0-100 |
| source | string | 是 | 数据来源，如 Polymarket |
| updatedAt | string | 是 | 更新时间，ISO 8601 |

← 09 §4.5 响应 DTO

### 1.5 FundNav（基金净值）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| fundCode | string | 是 | 基金代码 |
| navDate | string | 是 | 净值日期，格式 YYYY-MM-DD |
| accumulatedNav | number | 是 | 累计净值 |
| dailyGrowth | number | 否 | 日增长率(%) |

← 09 §4.6 响应 DTO, 参考代码 fund_nav.sql

## 2. 存储格式

### 2.1 文件结构

```
data/
├── timeline.json       # 时间轴数据（周节点列表）
├── thematic_funds.json # 主题基金数据
├── small_scale_funds.json # 小规模基金数据
└── situation_cache.json # 局势分值缓存
```

### 2.2 timeline.json

```json
{
  "weeks": [
    {
      "weekNumber": 1,
      "startDate": "2026-03-01",
      "endDate": "2026-03-07",
      "situationScore": 65,
      "summary": "美伊冲突升级，双方互相指责",
      "createdAt": "2026-04-15T10:00:00Z",
      "updatedAt": "2026-04-15T10:00:00Z"
    }
  ]
}
```

### 2.3 thematic_funds.json

```json
{
  "funds": [
    {
      "theme": "原油",
      "fundName": "南方原油A",
      "fundCode": "501018",
      "returnRate": 5.2,
      "weekNumber": 1
    }
  ]
}
```

### 2.4 small_scale_funds.json

```json
{
  "funds": [
    {
```

## 3. Storage 方法清单

### 3.1 TimelineStorage

| 方法名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| getAllWeeks | - | WeekNode[] | 获取所有周节点 |
| getWeekByNumber | weekNumber: int | WeekNode | 获取指定周节点 |
| saveWeek | week: WeekNode | void | 保存周节点 |
| updateWeek | week: WeekNode | void | 更新周节点 |

### 3.2 FundStorage

| 方法名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| getThematicFunds | weekNumber: int | ThematicFund[] | 获取指定周主题基金 |
| getSmallScaleFunds | weekNumber: int, limit: int | SmallScaleFund[] | 获取指定周小规模基金 |
| saveThematicFund | fund: ThematicFund | void | 保存主题基金 |
| saveSmallScaleFund | fund: SmallScaleFund | void | 保存小规模基金 |
| checkOverlap | fundName: string | boolean | 检查基金是否已存在于主题基金中 |

### 3.3 SituationStorage

| 方法名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| getCurrentScore | - | SituationData | 获取当前局势分值 |
| saveScore | score: SituationData | void | 保存局势分值 |
| isCacheValid | - | boolean | 检查缓存是否有效（24小时内） |

### 3.4 FundNavStorage

| 方法名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| getNavByFundCode | fundCode: string, period: string | FundNav[] | 获取指定基金的净值数据 |
| getNavByDateRange | fundCode: string, startDate: string, endDate: string | FundNav[] | 按日期范围获取净值 |
| saveNav | nav: FundNav | void | 保存净值数据 |
| fundExists | fundCode: string | boolean | 检查基金是否存在 |

← 参考代码 calculate_fund_performance.py

## 4. 数据约束

| 约束 | 说明 |
|------|------|
| 周编号唯一 | weekNumber 必须唯一 |
| 基金去重 | 小规模基金不能与主题基金重叠（按 fundName） |
| 规模限制 | 小规模基金 scale 必须 < 5 |
| 分值范围 | situationScore 必须在 0-100 之间 |
